# Security Policy

Do not report security issues through a public GitHub issue.

For security-sensitive reports, use the private contact channel that the CostGuard
publisher lists in release notes, commercial support terms, or the project website.

When reporting a vulnerability, include:

- affected CostGuard version;
- operating system and architecture;
- whether OpenClaw was running locally or remotely;
- steps to reproduce;
- expected impact;
- whether any sensitive data may have been exposed.

If you are preparing this repository for public launch, replace this file with a
named security contact before publishing broadly.
